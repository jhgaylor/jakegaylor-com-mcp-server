Guide for connecting your AI assistant to Me

Welcome! This is an Web server that exposes this website along with an MCP (Model Context Protocol) server that lets AI assistants learn about me.

What is MCP?

Model Context Protocol (MCP) is a standardized way for AI models to communicate with external services to request information or execute actions, enabling them to stay up-to-date and perform tasks beyond their training data.

How to Use

AI assistants can connect to this server at the /mcp endpoint to learn more about me and my work.

Connection Instructions

For Claude

Add this configuration to your Claude config:

{
  "mcpServers": {
    "jake-gaylor-mcp": {
      "command": "npx",
      "args": ["-y", "@jhgaylor/me-mcp"]
    }
  }
}

This will automatically install and run the MCP server locally when needed.

For Cursor

Add this configuration to your Cursor settings:

  1. Open Cursor Settings from the Command Palette (Command+Shift+P or Ctrl+Shift+P)
  2. Find "MCP Servers" setting
  3. Add the following configuration:
{
  "mcpServers": {
    "jake-gaylor-mcp": {
      "command": "npx",
      "args": [
        "-y",
        "@jhgaylor/me-mcp"
      ]
    }
  }
}

For Windsurf

Configure Windsurf to connect:

  1. Go to Windsurf Settings
  2. Navigate to the "AI Assistants" section
  3. Find "MCP Servers" option
  4. Add the following configuration:
{
  "mcpServers": {
    "jake-gaylor-mcp": {
      "command": "npx",
      "args": [
        "-y",
        "@jhgaylor/me-mcp"
      ]
    }
  }
}

For Zed

Set up Zed to connect:

  1. Open Zed settings.json (Cmd+, or Ctrl+,)
  2. Add the following MCP server configuration:
"context_servers": {
  "jake-gaylor-mcp": {
    "settings": {},
    "command": {
      "path": "npx",
      "args": ["-y", "@jhgaylor/me-mcp"]
    }
  }
}

Save the file and restart Zed if necessary

MCP Documentation

Resources

This MCP server provides the following resources:

  • candidate-info://resume-text: Resume content as text
  • candidate-info://resume-url: URL to the resume
  • candidate-info://linkedin-url: LinkedIn profile URL
  • candidate-info://github-url: GitHub profile URL
  • candidate-info://website-url: Personal website URL
  • candidate-info://website-text: Content from the personal website

Tools

This MCP server also provides tools that return candidate information:

  • get_resume_text: Returns the candidate's resume content as text
  • get_resume_url: Returns the URL to the candidate's resume
  • get_linkedin_url: Returns the candidate's LinkedIn profile URL
  • get_github_url: Returns the candidate's GitHub profile URL
  • get_website_url: Returns the candidate's personal website URL
  • get_website_text: Returns the content from the candidate's personal website

Example Usage (TypeScript)

import { Client } from "@modelcontextprotocol/sdk/client/index.js";
import { StreamableHTTPClientTransport } from "@modelcontextprotocol/sdk/client/streamableHttp.js";

async function connectToMCP() {
  const baseUrl = new URL("https://ai.jakegaylor.com/mcp");
  
  try {
    // Initialize the client
    const client = new Client({
      name: 'example-mcp-client',
      version: '1.0.0'
    });
    
    // Connect using StreamableHTTP transport
    const transport = new StreamableHTTPClientTransport(baseUrl);
    await client.connect(transport);
    
    console.log("Connected to MCP server!");
    
    // Access a resource
    const resumeResource = await client.getResource("candidate-info://resume-text");
    console.log("Resume:", resumeResource.contents[0].text);
    
    // Use a tool
    const githubUrl = await client.callTool({ 
      toolName: "get_github_url", 
      args: {} 
    });
    console.log("GitHub URL:", githubUrl.content[0].text);
    
  } catch (error) {
    console.error("Failed to connect to MCP server:", error);
  }
}

// Run the example
connectToMCP();
